Лабораторная работа 5. Потоковая обработка и работа с real-time данными. Вариант 6

Требования:
Latency < 200 мс для инференса
Accuracy > 85%
Обработка неструктурированных данных
Уникальное требование: p99 < 30 мс

Чтобы запустить проект нужно перейти  в католог и выполнить команду: "docker-compose up"

Выводы:
Для выполнения уникального требования p99 < 30 мс необходимо использовать Apache Flink
Благодаря архитектуре true-streaming обработки Flink обеспечивает детерминированно низкую задержку, что критически важно для медицинских систем реального времени.

Watermark стратегия успешно обрабатывает late-arriving медицинские данные
Система корректно обрабатывает 9.09% опоздавших записей пациентов, обеспечивая целостность анализа рисков даже при нестабильной доставке данных.

State management позволяет отслеживать историю состояний пациентов
Благодаря TTL State в 24 часа система сохраняет контекст по каждому пациенту, что значительно улучшает точность прогнозирования рисков повторной госпитализации.
